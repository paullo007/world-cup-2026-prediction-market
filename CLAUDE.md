## What it is
A Polymarket-style **play-money** prediction market for the FIFA World Cup 2026
(Jun 11 ‚Äì Jul 19, 2026). Binary YES/NO markets priced by an LMSR automated market maker.
New accounts start with 10,000 coins (raised from 2,000 in Session 21, from 1,000 in Session 15); winning shares pay 1 coin on admin resolution.

- **Live:** https://world-cup-2026-prediction-market.vercel.app
- **Repo:** https://github.com/paullo007/world-cup-2026-prediction-market (public)
- **Local (code root):** `‚Ä¶/08_WORLD CUP PREDICTION MARKET/World Cup 2026 Prediction Market`

## Stack
Next.js 14 (App Router) ¬∑ TypeScript ¬∑ Tailwind ¬∑ Prisma ¬∑ PostgreSQL on **Supabase
(region ap-southeast-1, Singapore)** ¬∑ NextAuth (**two** credential providers ‚Äî see Auth)
¬∑ hosted on **Vercel** (GitHub auto-deploy on `main`).

## How to run / test
```bash
npm install
npm run dev        # http://localhost:3000
npm run build      # prisma generate && next build  (use to catch type/build errors)
npm run lint
npm run db:push    # push prisma schema to DB
npm run db:seed    # tsx prisma/seed.ts ‚Äî admin user + markets
```
`.env` (gitignored) holds `DATABASE_URL` (pooled, 6543, pgbouncer), `DIRECT_URL` (5432),
`NEXTAUTH_URL`, `NEXTAUTH_SECRET`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`. Seed refuses to run
without `ADMIN_PASSWORD`. Results pipeline adds `CRON_SECRET` (protects the cron route;
Vercel sends it as a Bearer token) and `THESPORTSDB_API_KEY` (free key "3" default; ESPN
needs no key). Set `CRON_SECRET` + `THESPORTSDB_API_KEY` in Vercel env too.

## Conventions / hard rules
- **Never `git commit` / `git push`, and never write to the live DB, without Paul's explicit OK.** "go" / "proceed" means *start coding only* ‚Äî push is a separate approval. (Memory `feedback_wait_for_approval_before_push`.)
- **Vercel + Supabase CLIs: use access tokens, not browser device-flow** (device-flow won't persist on this Mac). (Memory `feedback_vercel_supabase_cli_use_tokens`.)
- **FIFA times/venues come from Paul's screenshots** ‚Äî FIFA's site is a JS SPA (empty to fetcher), aggregators hallucinate venues. Screenshots are the source of truth.
- Vercel auto-deploy occasionally **misses a webhook** ‚Üí re-trigger with an empty commit.
- ‚õî **DB transactions must be pooler-safe (Session 11).** Prisma **interactive** transactions (`db.$transaction(async (tx) => ‚Ä¶)`) fail *intermittently* over Supabase's transaction-mode pooler (pgbouncer, the 6543 `DATABASE_URL`) with `Transaction API error: Transaction not found ‚Ä¶ obtained before disconnecting` (P2028). This silently broke ALL match resolution for weeks (every `resolveMatchGroup` threw; matches stuck "Closed ‚Äî awaiting result"). Rules: **resolution uses the batched `$transaction([...])` array form** (pre-read, then one atomic non-interactive batch ‚Äî `resolveMarketOps()` in `lib/trade.ts`); **every transaction (trade + resolve) is wrapped in `withTxRetry()`** (retries P2028/connection-drop, never a `TradeError`). When retrying a batched transaction, **rebuild the ops each attempt** ‚Äî a `PrismaPromise` is single-use. Don't reintroduce a long interactive transaction on the hot path.
- **Never depend on Vercel Cron here (Session 11).** The `vercel.json` cron (`/api/cron/results`) was **never enabled** on this plan (`crons: []`, `enabledAt: null`) ‚Äî it had not fired once. Result settling now has THREE independent, idempotent triggers: on-view self-heal (`<AutoResolve>` ‚Üí throttled `/api/auto-resolve`), a **GitHub Actions** cron every 15 min (`.github/workflows/resolve-results.yml` ‚Üí `/api/auto-resolve`, no secret), and the public manual button. The `vercel.json` cron is left as a harmless extra backstop.
- Verify carefully: `grep | head` masks grep's exit code; React `<!-- -->` markers split text ‚Äî both have caused false positives.
- **ESPN scoring-play types: penalties are "Penalty - Scored" (no "goal").** Parse goals with `/goal|penalty/i`, not `/goal/i`, or penalty goals vanish; own goals are KEPT (credited to the benefiting team). The ingest **reconciliation self-check** (`scorers == homeGoals+awayGoals`) now catches any such gap automatically (`‚öÝ scorers N/M` in logs + `resultDetail`) ‚Äî don't remove it. Changes to scorer parsing must keep `npm run test:scorers` green. **Own goals are stored AS scorers (credited to the benefiting `team`, but carrying the opponent player's `name`+own-team flag) so the score reconciles ‚Äî yet they must be EXCLUDED from player goal tallies** (Goals tab, top scorers): they aren't the named player's achievement and would mis-attribute them to the wrong country (fixed Session 21; `if (s.ownGoal) continue;` in `app/goals/page.tsx`). `goalsForRoster` is already safe (matches scorer.team to the country). Guarded by the own-goal-conservation check in `npm run test:played-matches`.
- **Scorer names go through `cleanPlayerName()` (Session 13).** ESPN builds `fullName` as "first last", so a mononym with a null last name arrives as e.g. "Tr√©z√©guet null". `parseEspnScorers` runs `cleanPlayerName` (strip standalone null/undefined tokens, collapse whitespace, fall back to `displayName`). Don't bypass it. Stored data was backfilled via `scripts/backfill-scorer-names.ts`.
- ‚õî **Live scores are DISPLAY-ONLY and MUST stay separate from settlement (Session 13).** `lib/liveScores.ts` deliberately keeps in-progress matches (state "in"/"post") that the resolution path discards. Nothing in the live path (`/api/live-scores`, `LiveScoreProvider`, `MatchCard3Way` LIVE branch, `LiveNowStrip`) may resolve a market or pay out ‚Äî settlement still fires **only on `completed=true`** via the ingest pipeline. Never wire live partial scores into `resolveMatchGroup`/`ingestAndPublish`.
- ‚õî **Bracket auto-fill is DISPLAY-ONLY too (Session 14).** `lib/bracketSync.ts` fills team names into the bracket from ESPN; it must never resolve a market or pay out. The `EVENT_TO_MATCH` id-map is the source of truth for ESPN‚Üíour-slot ‚Äî **R32 is verified; re-verify R16/QF/SF/Final ids before each round populates** (teams are TBD until then, so a wrong later-round id shows nothing now but would mis-place teams in July). Only confirmed real teams (`ALL_TEAMS`) are written; placeholders stay as positional labels. Manual `BracketAssignment` rows override ESPN.
- **Changing scorer/assist parsing needs a one-off re-backfill of resolved matches** ‚Äî resolution is idempotent, so the live pipeline won't rewrite already-RESOLVED markets' `scorers`. Pattern: `fetchEspn(12)` ‚Üí match by `pairKey` ‚Üí rebuild scorers (+`attachAssists`) ‚Üí update `Market.scorers` (dry-run first, then `--apply`; live-DB write, needs Paul's OK).
